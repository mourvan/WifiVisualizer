Операционная система: Ubuntu x64

Оборудование: Intel Core i5 2 ядра (1.8 ГГц), 8 ГБ памяти

Компиляция:
вызов следующей команды из папки проекта (mg3)
g++ -O2 -fopenmp -o main src/Main.cpp src/Scene.cpp src/Tracer.cpp src/EasyBMP/EasyBMP.cpp src/TinyObjLoader/tiny_obj_loader.cc

Формат файла настроек:
1 строка - разрешение камеры (2 значения через пробел)
2 строка - путь к файлу .obj
3 строка - трехмерные координаты роутера (через пробел)
4 строка - мощность сигнала
5 строка - трехмерные координаты камеры (через пробел)
6 строка - направление камеры (3 координаты вектора ориентации через пробел, можно не нормализованные)
7 строка - вектор наклона камеры (вектор “вверх” камеры, определяет угол наклона “головы”
8 строка - углы обзора по горизонтали и по вертикали соответственно. Рекомендуется 90 и 74
9 строка - Если TRUE то постобработка автоконтрастом (параметр autocontrast). Любая другая  непустая строка - без обработки.
10 строка- Если TRUE то сглаживание вокселей медианной фильтрацией. Любая другая непустая cтрока - обычный бокс фильтр.


Рекомендуемые параметры указаны в конфиг файлах config1.txt (вид сверху) и config2.txt (перспектива)

Время работы программы: 
config1 - 10c
config2 - 25c
config3 - 20c

Дополнительная часть: 
1) Текстуры стен и пола. Реализована модель освещения Фонга. Источник света - камера.
2) Сложные полигональные объекты. Программе можно “скормить” помещение со сложными объектами, так как реализована загрузка и проверка на пересечение со всеми полигонами модели, но время обработки соответственно сильно возрастет.
3) Пользовательский интерфейс. Файл с настройками указывается первым аргументом программы. В нем можно прописать все параметры. 
4) Параллелизм. Простая реализация трассировки лучей из антенны, трассировки лучей из камеры, сохранения изображения в bmp на OpenMP
5) Более сложный фильтр сглаживания воксельной сетки - медианная фильтрация
6) Постобработка результата - автоконтраст
